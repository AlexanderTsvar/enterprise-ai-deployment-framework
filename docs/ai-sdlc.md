# AI SDLC Overview

Traditional SDLC validates whether software works as intended. AI SDLC must also validate whether the system behaves safely, responsibly, and within acceptable risk boundaries.

## Traditional SDLC

```text
Requirements → Development → QA → UAT → Deployment → Support
```

## AI SDLC

```text
Business Need → AI Product Design → Risk Assessment → Guardrails → Evaluations → Red Teaming → Deployment Gate → Observability → Continuous Validation
```

## Key differences

- AI behavior is probabilistic, not fully deterministic.
- Functional success does not always equal deployment readiness.
- AI systems require risk-based validation before launch.
- Guardrails must be tested, not assumed.
- Production monitoring must include behavioral and safety signals.
